1. Class
Definition: A class is a blueprint or a template used to create objects. It defines the properties (data variables) and methods (functions) that the objects created from it will possess.

Analogy: It is like a house blueprint; it describes the structure, but it is not the actual house.

2. Constructor
Definition: A constructor is a special built-in function inside a class that executes automatically whenever a new object is instantiated using the new keyword.

Purpose: Its primary job is to initialize the object's properties with values at the moment of creation.

💻 Standard Approach Example (The Long Way)
In this approach, properties are declared at the top of the class (making them public by default) and mapped manually inside the constructor using the this keyword.

TypeScript
class BottleMaker {
    // 1. Declare properties at the top (Public by default)
    color: string;
    price: number;

    // 2. Accept arguments in the constructor
    constructor(color: string, price: number) {
        // 3. Manually map values to the permanent object memory
        this.color = color;
        this.price = price;
    }
}

// Creating an object instance
let b1 = new BottleMaker("blue", 25);
console.log(b1); // Output: BottleMaker { color: 'blue', price: 25 }
📌 CRITICAL NOTE: The Parameter Property Mechanics
The Rule: Constructor parameters are naturally just temporary local variables. They only live inside the constructor's curly braces {} and disappear completely once the execution finishes. To save them permanently onto the object instance, they must either be explicitly mapped using this or utilize a TypeScript access modifier shortcut.

Approach Comparison for Notes:
Without Access Modifier (color: string): Treated strictly as a local function argument. It will not be saved onto the resulting object instance unless you explicitly write this.color = color. Printing the object will return an empty shape {} if no assignment is made.

With Access Modifier (public color: string): Activating TypeScript's Parameter Property Shortcut. Placing public, private, or protected directly inside the constructor arguments tells the compiler to automatically declare the variable at the top of the class and automatically inject the this.color = color assignment behind the scenes.

Shortcut Example:
TypeScript
class CompactBottleMaker {
    // Skips top declarations and skips 'this' assignments entirely
    constructor(public color: string, public price: number) {}
}

let b2 = new CompactBottleMaker("green", 30);
console.log(b2); // Output: CompactBottleMaker { color: 'green', price: 30 }

🔑 What is the this Keyword?
Definition: In JavaScript and TypeScript, this is a special reference keyword that points to an execution context object. It does not have a fixed value; its value is dynamically determined by how a function is called, not where it is defined.

Core Purpose: It allows methods and functions to reuse code configurations across different object instances by dynamically accessing properties belonging to the current owner object.

📜 The 4 Rules of Context Binding
The value of this changes depending on the execution context. Here are the four scenarios:

1. Global Context (Default Binding)
If you reference this outside of any function or class, it defaults to the global environment container.

In a browser console: this points to the window object.

In Node.js: this points to the global module.exports object.

2. Method Context (Implicit Binding)
When a function is executed as a method inside an object (using dot notation like obj.method()), this points directly to the object to the left of the dot.

TypeScript
const employeeProfile = {
    username: "Dharmendra",
    greetUser: function() {
        // 'this' points directly to employeeProfile
        console.log(`Welcome back, ${this.username}`);
    }
};

employeeProfile.greetUser(); // Output: Welcome back, Dharmendra
3. Class/Constructor Context (New Binding)
When a function is called with the new keyword or executed inside an Object-Oriented Class constructor, this points to the brand-new empty object instance currently being allocated in memory.

TypeScript
class SDE {
    constructor(public role: string) {
        // 'this' points to the new blank instance created by 'new'
        this.role = role; 
    }
}

const dev1 = new SDE("MERN Stack Developer");
console.log(dev1.role); // Output: MERN Stack Developer
4. Explicit Context Binding (call, apply, bind)
If you want to force a function to use a specific object as its context, you can explicitly hijack and bind the context using call(), apply(), or bind().

TypeScript
function printSalary() {
    console.log(`Current compensation: ${this.salary}`);
}

const accountA = { salary: 85000 };
const accountB = { salary: 95000 };

// Forcing printSalary to use accountA, then accountB as its context
printSalary.call(accountA); // Output: Current compensation: 85000
printSalary.call(accountB); // Output: Current compensation: 95000
⚠️ The Arrow Function Exception
Regular functions track context dynamically based on who calls them. Arrow functions do not have their own this keyword. Instead, an arrow function inherits the this value from its surrounding, parent lexical scope at the exact moment it is defined.

TypeScript
const dashboard = {
    title: "Admin Panel",
    
    // Regular function: gets its context from the caller (dashboard)
    showTitleNormal: function() {
        console.log(this.title); // ✅ Output: "Admin Panel"
    },
    
    // Arrow function: captures context from the global environment
    showTitleArrow: () => {
        console.log(this.title); // ❌ Output: undefined (or global scope)
    }
};
📌 Key Rule for Notes: Use regular functions/methods inside your objects and class models when you need to interact with properties via this. Avoid using arrow functions as primary object methods, as they will detach from the object instance and break context binding.

🔒 What are Access Modifiers?Definition: Access Modifiers are keywords in TypeScript that control the visibility and accessibility of class properties (variables) and methods (functions) from outside the class.Core Purpose: They implement Encapsulation (the first pillar of OOP) by protecting internal object data from unauthorized modification or access.JavaScript Compilation Behavior: Access modifiers are purely a TypeScript development-time feature. JavaScript does not natively enforce them, so they are completely stripped away during compilation.🛠️ The 3 Primary Access ModifiersTypeScript provides three main visibility modifiers. If you do not specify a modifier before a property or method, TypeScript defaults it to public.1. public (Accessible Everywhere)Properties and methods marked public can be accessed from inside the class, inside child classes, and outside by any external object instances.This is the default setting.2. private (Accessible Only Inside the Same Class)Properties and methods marked private are strictly locked inside the specific class that declared them.They cannot be accessed from external code, and they cannot be accessed by inherited child classes.3. protected (Accessible Inside Class and Child Classes)Properties and methods marked protected act like a middle ground.They are locked away from the external world, but child classes that extend the parent class can access them freely.💻 Code Example: Implementing All ModifiersHere is how all three access levels operate within an inheritance chain:TypeScript// Parent Class
class CompanyAsset {
    public assetName: string;       // Open to everyone
    private financialValue: number;  // 🔒 Hidden from everyone except CompanyAsset
    protected securityCode: string;  // 🔑 Hidden from outside, but open to Child Classes

    constructor(name: string, value: number, code: string) {
        this.assetName = name;
        this.financialValue = value;
        this.securityCode = code;
    }

    public logValue(): void {
        // ✅ Private properties can be read easily inside their own class
        console.log(`Value of ${this.assetName} is INR ${this.financialValue}`);
    }
}

// Child Class inheriting from CompanyAsset
class ServerAsset extends CompanyAsset {
    public ipAddress: string;

    constructor(name: string, value: number, code: string, ip: string) {
        super(name, value, code);
        this.ipAddress = ip;
    }

    public accessSystem(): void {
        console.log(`Connecting to ${this.assetName}...`); // ✅ Allowed (Public)
        console.log(`Security Code: ${this.securityCode}`); // ✅ Allowed (Protected)
        
        // ❌ TS Compiler Error: Property 'financialValue' is private and only accessible within class 'CompanyAsset'.
        // console.log(this.financialValue); 
    }
}

// --- Execution and Verification ---
const mainServer = new ServerAsset("Noida Mainframe", 500000, "SEC_9921", "192.168.1.1");

console.log(mainServer.assetName); // ✅ Allowed (Public) - Outputs: Noida Mainframe

// ❌ TS Compiler Error: Property 'securityCode' is protected and only accessible within class 'CompanyAsset' and its subclasses.
// console.log(mainServer.securityCode); 

// ❌ TS Compiler Error: Property 'financialValue' is private and only accessible within class 'CompanyAsset'.
// console.log(mainServer.financialValue); 
🚀 Shortcut: Using Modifiers in Parameter PropertiesAs we learned earlier, you can skip declaring properties at the top of the class entirely by placing access modifiers directly inside the constructor arguments:TypeScriptclass ShortFormUser {
    // TypeScript automatically creates 'username' as public and 'token' as private
    constructor(public username: string, private token: string) {}

const user = new ShortFormUser("Dharmendra", "JWT_SECRET_XYZ");

Bhai, aapne ek baar phir bilkul sahi point pakda hai. Yeh wahi same problem hai jo humne private keyword ke sath dekhi thi: TypeScript ka readonly keyword bhi sirf ek design-time check hai. Jab tsc is code ko JavaScript me compile karta hai, toh readonly keyword poori tarah se delete ho jata hai. JavaScript engine ke liye this.fname ek normal variable ban jata hai, isiliye console me value change ho jati hai.

Isko runtime (JS engine level) par permanent protect karne ke do sabse solid tareeqe hain:

Method 1: JavaScript Native Getter Use Karein (Most Professional)
Agar aap kisi property ka sirf get method banate hain aur uska set method nahi banate, toh JavaScript runtime level par usko automatic ek Read-Only property bana deta hai. Phir koi bhi use console me change nahi kar payega.

TypeScript
class UserProfile {
    // 1. Property ko private ya # se hide karein
    private _fname: string; 

    constructor(fname: string) {
        this._fname = fname;
    }

    // 2. Sirf Getter banayein (No Setter)
    public get fname(): string {
        return this._fname;
    }

    changeName(newName: string) {
        // ❌ VS Code bhi error dega aur Node.js runtime bhi isko change nahi karne dega
        // this.fname = "Doe"; 
    }
}

const user = new UserProfile("Dharmendra");
console.log(user.fname); // ✅ Read works: "Dharmendra"

// ❌ Runtime Error: Cannot set property fname of #<UserProfile> which has only a getter
// user.fname = "Rahul"; 
Method 2: Constructor me Object.freeze() ka use karein
Agar aap chahte hain ki class bante hi uske andar ki saari properties hamesha ke liye lock ho jayein (taaki class ke andar ka koi method bhi use change na kar paye), toh aap constructor ke aakhri me Object.freeze(this) ka use kar sakte hain.

TypeScript
class UserProfile {
    constructor(public readonly fname: string) {
        // 🔒 Yeh line pure object ke properties ko runtime par freeze kar degi
        Object.freeze(this); 
    }

    changeName(newName: string) {
        this.fname = "Doe"; // ❌ TS Error dikhayega
    }
}

const user = new UserProfile("Dharmendra");

// ❌ JS Strict mode me yeh line runtime par error degi aur value change nahi hone degi
user.changeName("Doe"); 
console.log(user.fname); // Keval "Dharmendra" hi rahega
Notes Summary for Revision:
readonly keyword sirf TypeScript compiler tak mehdood hai. JS file me badalte hi yeh gayab ho jata hai.

Runtime level par values ko freeze rakhne ke liye Getters (without setters) ya Object.freeze(this) ka use kiya jata hai.

A Static Member (property or method) in TypeScript and JavaScript belongs to the class itself, rather than to any specific instance or object created from that class.

📘 Definition Note
Instance Member (Normal): Every time you run new ClassName(), a fresh copy of that property is created in memory for that specific object.

Static Member: Only one single copy exists in memory, tied directly to the blueprint (the class). It is shared across the entire application.

Key Feature: You access static properties and methods directly using the Class Name, without creating an object instance via new.

💻 Code Example: Implementing Static Members
Imagine you want to keep track of how many total users are registered in your Employee Management system. You don't want every employee object to have its own independent counter; you need a single global counter.

TypeScript
class Employee {
    // 1. Static Property: Lives on the class, tracks global count
    public static totalEmployees: number = 0;

    // 2. Instance Property: Every employee gets their own unique name
    public name: string;

    constructor(name: string) {
        this.name = name;
        
        // Increment the global counter every time a new employee is created
        // Note: We use "Employee.totalEmployees", NOT "this.totalEmployees"
        Employee.totalEmployees++;
    }

    // 3. Static Method: Utility helper belonging strictly to the class
    public static getCompanyInfo(): string {
        return "System Status: Active. Managing enterprise workforce.";
    }
}

// --- Verification Actions ---

// Call the static method directly on the Class without using 'new'
console.log(Employee.getCompanyInfo()); 
// Output: System Status: Active. Managing enterprise workforce.

// Create instances
const emp1 = new Employee("Dharmendra");
const emp2 = new Employee("Rahul");

// Check the shared data pool
console.log(emp1.name); // Output: Dharmendra
console.log(emp2.name); // Output: Rahul

// Check the static property via the class reference
console.log(Employee.totalEmployees); // Output: 2

// ❌ TS Compiler Error: Property 'totalEmployees' is a static member of type 'Employee'
// console.log(emp1.totalEmployees); 
📌 Why and When to Use Static Members?
For Counters and Global Trackers: Like counting active database connections, tracking active network sockets, or managing cache hits.

For Utility/Helper Functions: When a function does not need any unique data from an object instance to perform its calculation. For example, JavaScript's native Math.abs(-5) or Math.ceil(4.2) are static methods on the global Math class.

Quick Summary for Notes:
The Static Rule: Static members are declared using the static keyword. They are allocated once in memory under the namespace of the Class itself. They cannot access instance properties using this because they execute outside the context of any individual object creation.

An Abstract Class is a restricted class that cannot be used to create objects directly. It exists solely to act as a parent blueprint for other classes to inherit from.

Think of it as a half-built template: it defines a common structure and forces child classes to implement specific missing details.

⚡ Simple Definition
Definition: An abstract class is a base class declared with the abstract keyword. It serves as an incomplete blueprint for inheritance.

Core Rule: You cannot instantiate it directly using the new keyword. It can only be extended by subclasses (extends).

Purpose: It provides a mix of fully implemented methods (shared code) and unimplemented methods (forced code templates).

🧩 The Two Types of Members Inside an Abstract Class
An abstract class is powerful because it can contain both completed and uncompleted parts:

Abstract Methods: Declared without a body (no {}). They only define the function signature (name, parameters, and return type). Every child class must write the actual implementation for these methods.

Regular (Concrete) Methods: Fully written functions with a body {}. Child classes inherit these automatically and can use them right away without rewriting them.

💻 Code Example: Database Controller Template
Imagine you are building your Employee Management Project backend. You want a standard structure for all database handlers (like MongoDB, Postgres, or a Mock DB for testing).

TypeScript
// 1. Defining the Abstract Base Template
abstract class DatabaseController {
    // Regular Method: Shared config code used by all databases
    public connect(): void {
        console.log("Establishing secure connection to the server cluster...");
    }

    // Abstract Methods: No body! Child classes MUST define how to read/write data.
    public abstract saveUser(userId: string, data: object): void;
    public abstract deleteUser(userId: string): boolean;
}

// ❌ TS Compiler Error: Cannot create an instance of an abstract class.
// const baseDb = new DatabaseController(); 

// 2. Concrete Subclass: Implementing the required blueprint
class MongoDatabase extends DatabaseController {
    
    // You must implement the abstract method 'saveUser' or TS will throw an error
    public saveUser(userId: string, data: object): void {
        console.log(`[MongoDB Engine] Writing user data for ID: ${userId} into collection.`);
    }

    // You must implement the abstract method 'deleteUser'
    public deleteUser(userId: string): boolean {
        console.log(`[MongoDB Engine] Removing document matching ID: ${userId}.`);
        return true;
    }
}

// --- Execution ---
const productionDb = new MongoDatabase();
productionDb.connect();               // ✅ Works (Inherited directly from abstract parent)
productionDb.saveUser("EMP_101", {}); // ✅ Works (Implemented by child class)
